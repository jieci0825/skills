# 输出节奏参考

## 简单需求

适合信息明确、范围小、影响面有限的需求。直接一次性给出完整方案，重点在效率。

## 中复杂需求

适合存在一定耦合度，但还不需要拆成很多轮讨论的需求。先统一需求与方案，再单独输出 MVP 与迭代。

## 高复杂需求

适合系统性、探索性或影响面大的需求。严格分步推进，上一轮结论应成为下一轮输入，避免一次性给出过多内容。

## 建议话术

### 中复杂需求

```md
当前需求建议分两阶段推进：
第一阶段先统一需求理解与方案方向；
第二阶段再收敛 MVP 范围和后续迭代路线。
```

### 高复杂需求

```md
当前需求复杂度较高，建议按 step-by-step 推进。
本轮先完成 Step 1：需求确认；
确认后再进入主流方案对比。
```
